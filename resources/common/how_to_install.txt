# installation instructions 


**IMPORTANT READ THIS CAREFULLY, AND FOLLOW PRECISELY** 
failing may mean your ssp/xmx may not boot, and may have to reinstall the factory image!



note: the following may seem 'daunting' or a lot of info, but its not really.
frankly it boils down to copy files over, rename 2 files, reboot ;) 

BUT, taking it slowly and carefully, could save some headaches.
also the majority of this doc, is just 'useful information'


## PART ONE - install TB APPS and plugin


a) download  tbapps zip file for YOUR PLATFORM
ensure you download the XMX or SSP version as appropriate (!)

b) remove the sdcard from ssp/xmx , place in your pc/mac

c) unzip the zip file and copy contents to your sdcard

note: some unzip tools might create a directory, you want the contents of that directory.

overwrite files as necessary.

e.g. when you have copied, you should have perccmd.json in the ROOT of the sdcard, check this.
you should also see a file called 'install' in the root — this is used later in Part Three.


NOTE: if a tbplugins release has been made SINCE the tbapp release, you can use the updated plugins


d) eject sdcard from computer, and place back in ssp/xmx


## PART TWO - test the plugins

this is just a sanity check that all is good...

powere up ssp/xmx, 
synthor will start
check the CLDS plugin is present in the module browser (if present, all plugins are installed correctly).



## PART THREE - installing bootstrap so you can run standalone programs


THIS ONLY EVER NEEDS TO BE DONE ONCE !! (unless instructed otherwise ;) )
even if installing fresh versions of the apps etc.


basically, this step is required to just change what the SSP/XMX boots by default.
once its done, it'll always boot into PercCmd, which will handle launching the right application.


a) power down ssp/xmx, remove sd card
b) place sdcard into your computer


next steps are VERY important to get name correctly 


c) copy SYNTHOR to you desktop (as a backup)
d) rename SYNTHOR (on sdcard) to SYNTHOR.ORIG
e) rename install to SYNTHOR 

f) place your sd card back into SSP/XMX


g) turn on the SSP/XMX
now when you power on, it'll take a bit longer as it updates config, then reboots back into synthor.
(about 30-45 seconds), just wait.

Note: on the SSP its quite possible/likely it wont reboot rather just stay on a black screen.
this is fine... SSP doesnt like doing a graceful power down..
if after 90 seconds , its not appeared... just powercyle



CONGRATULATIONS YOU ARE DONE !!!!

the install is complete. the sections below explain how to use PercCmd and contain troubleshooting info if needed — but most users won't need them.

now we can test it.


once it reboots or your powercycle you should end up back in Synthor 'as normal'


however, now we have PercCmd installed, which means when you powercycle your SSP/XMX
if you hold down a 'softkey' , 1-8 it will launch an application

which application is determined by the contents of perccmd.json.
so users/developers can add new ones.

Note: the last applicaton is 'saved', 
so next time you boot you will go back to that application without presssing any button.


the following apps are configured:
1 : Synthor
2 : Trax
3 : er301

1-4 = first 4 buttons, top row
5-8 = first 4 buttons, bottom row

example: 

we are in synthor (after install)

whilst powering up, hold button 3 down, until you see  er301
have a play :) 

(optional) power cycle , no buttons held, you will go back to er301


whilst powering up, hold button 1 down -  until you see synthor


(optional) power cycle , no buttons held, you will go back to synthor

-----------------------------------------------------------------------------
Trouble shooting - installation
-----------------------------------------------------------------------------


honestly, not a lot to go wrong

if you dont name install to SYNTHOR correctly, the install will do nothing.

if you didnt rename the synthor binary correctly, then that'll mean it wont boot
BUT you can just put back SYNTHOR on the card, and it'll start working.


Theres no real logging,
but you can see if the install ran by looking at the sdcard started
note case does not matter, as its a fat fs.

before running the install (i.e. after the renaming etc)
S80perccmd
SYNTHOR (< 1000 bytes, my install script)
SYNTHOR.ORIG (> 1 MB, real synthor)
NO, install (you renamed it ;)) 

after : powering up ssp/xmx, with the above
S80perccmd - no longer there, if install ran at all
SYNTHOR (> 1 MB) (real synthor)
install (< 1000 bytes, my install script, safe to remove now
SYNTHOR.ORIG - gone, it was renamed back to SYNTHOR


basically S80perccmd is 'gone', then the install was done

Trouble shooting - general
-----------------------------------------------------------------------------

This is designed to fallback to Synthor, if there are any issues.
if you have an issue you can delete "lastBtn", this will make it boot synthor again.

you can also rename perccmd.json, to (e.g.) perccmd.bak, if you have an issue with the config file.
if you have been editing it...

Background info - which may be helpful.
----------------------------------------------------------------------------

## what's the install doing?

on the factory image SYNTHOR is called when it booted up.
this (from a user perspective) is hardcoded...

so what I do, is replace this with something more flexible.

to do this, I need to run my own script (install).
we do this, by simply replacing (temporarily) SYNTHOR with my install script.
so when you power on, it runs my script.

the install script does 3 things (you can read it ;))

a) new boot procedure
it moves S80perccmd to /etc/init.d
this ensures perccmd.sh is now always called when the SSP/XMX is started

perccmd.sh launches our bootstrap program we use to intercept keys, and start what we want.


b) moves the install script (SYNTHOR) back to install

c) move SYNTHOR.ORIG back to SYNTHOR

so when you reboot, SYNTHOR is now back to the one you expect ;) 


note: I used to disable the synthor 'start' script in this install process..
I dont now, instead I disable it in S80perccmd.
this means, if for some reason S80perccmd is NOT installed correctly, it'll continue to boot synthor


## PercCmd
this process decides which process to start depending on which button is being held on the device.
(or uses the last process if none is held, lastBtn stores this)
this way we can start different processes at startup.
we can see in perccmd.json how these buttons are configured, so if you want you could have your own custom processes launched :) 



## could I uninstall this?
theoretically it be possible, as I don't delete anything during the install process.
basically I just rename the original start script
so to revert, we'd simply change perccmd.sh to 
a) mv /etc/init.d/D90synthor /etc/init.d/S90synthor , enable old startup
b) mv /etc/init.d/S80perccmd /etc/init.d/D80perccmd , disable new startup

however, its not really necessary.. as you PercCmd is just more flexible to leave in place,  and you can ignore if you dont use ;) 

SO... if you want to revert, then its better you just download the factory image and reinstall.

(I really give the above, just in cases devs how have a network connection to xmx/ssp want to mess with it )


easy to do if you can log into the SSP/XMX, 
if not you could do this in a script, but be careful, get it wrong, and it wont boot.
and you'll have to write a new SD card ;)
